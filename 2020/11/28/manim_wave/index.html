在网上查阅到的代码，提供了一个利用matpltlib提供散点坐标进行函数绘图的方法。
但如何让波形动起来呢？如果利用到之前写的DoubleWaveScene类，由于GraphScene提供的绘图方法是函数解析式绘图，即必须要传入一个函数。一种可行的方法是：修改self.graph_func，让它针对每一个不同的传入值x, t，返回一个对应相应的波形的函数值。具体函数设计如下。

fun1为\(x \leq 1\)部分的返回值函数，代表着左声道的传值。其中，self.time_series为存储时间的numpy数组，两个邻接数据的差为\(\frac{1}{sr}\)（\(sr\)为每秒取样率）；self.wave为存储波形的DataFrame(pandas中数据表类型)，其中数据的下标与self.time_series一一对应。
先调用self.search函数，在self.time_series中搜索返回第一个大于输入值\(x + 1000t\)的元素下标pos，再通过pos在self.wave中搜索相应的函数值。
func2的设计也是类似，只是把左声道变为了右声道，用来提供\(x \geq 1\)区间上的函数值。\((-1, 1)\)区间的返回值为0，作为一条水平的分界线。

  • 搜索算法第一步：

    其实搜索没必要在整个歌曲内进行。比如需求只是画出歌曲前10s的波形图，那就只用获取前10s的数据即可。

  • 在整个self.time_series数组中搜索一个数值耗时太久。但由于self.time_series是顺序（逐渐增加）的，因此可以考虑下使用一些顺序查找优化算法。

    • algorithm 1: 将整个区间分为多个小区间，先和每个小区间的临界值比较，确定数据具体在哪个小区间里，再在小区间里搜索具体的位置。（已实现并用到代码中，效果较好）这样能把成指数级（？）增长的算法时间复杂度降为线性级（？），即算法规模扩大仅仅体现在多搜索一个小区间而已上。
      小区间长度的选取：2s。选取原因是在运行时，渲染2s长度的视频速度最快。

这个优化算法真的写到自闭。。。
从原先第一版的数组存储方式改到numpy、pandas存储，读取和写入的时间果然变快了许多。使用pandas存储的时候犯了好多次错。
再到后面踌躇满志地写搜索算法，开始毫无思路，一直在想怎么改搜索函数（原先用的find_first()函数效率很低）；
后来在运行时偶然发现短视频渲染时间比较短（因为视频短数据量小，搜索的时间也就短），再进一步发现2s左右的短视频渲染效率最高，差不多能做到1s渲染1s的动画。于是萌生了分别渲染多个短时间视频，然后拼接起来的想法。奈何Manim封装得实在太好了，根本无从下手；
一遍遍看程序的过程中，发现了切入点：还是搜索函数，只要把搜索函数分成大段，先确定所处的大段，然后就只需要再所处的大段长度中渲染，就可以达到相似的效果，并且程序写得也更简洁更精炼；
不过能将一个原先渲染时长6+小时的视频优化到基本1s对1s，已经非常自豪了。
